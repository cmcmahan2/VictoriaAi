# Trend Report

Produce a weekly domain investing intelligence briefing for the App-Engine project.

## Format

Write a one-page briefing with exactly these five sections:

---

**WEEKLY DOMAIN INTELLIGENCE BRIEF**
Week of: [current date]

**1. TOP SIGNALS THIS WEEK**
Three to five specific named entities (companies, products, technologies) gaining commercial momentum. For each: name, why it matters for domain investing, velocity (rising/peak/declining).

**2. EMERGING NAMING PATTERNS**
What linguistic or structural patterns are appearing across multiple trends? (e.g., "X-as-a-service", "AI + [profession]", "[verb]base", "[noun]flow"). These predict what types of domains will be in demand 60–90 days out.

**3. DOMAINS TO WATCH**
Five specific domains (not registered yet, if possible) that align with this week's signals. For each: domain, basis trend, buyer profile, estimated value.

**4. COMPETITIVE INTELLIGENCE**
Any signals that major acquirers (Godaddy, Afternic, NameBio) or domain investors are already moving on. Avoid areas where you're too late.

**5. ACTION ITEMS**
Three concrete next steps for the portfolio. Specific domains to research, trends to monitor, or pipeline changes to make.

---

## Instructions

- Draw on any data available from the Finance MCP server (market trends, company signals) and the GitHub MCP server (trending repos often precede naming trends)
- Keep all recommendations specific — named entities only, no generic categories
- Tone: professional, direct, no filler. Read like a Goldman research note, not a blog post.
- Total length: fit on one page (400–600 words)
